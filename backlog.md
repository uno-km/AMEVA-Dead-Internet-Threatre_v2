# AMEVA 플랫폼 화 백로그 (Platformization Backlog)

본 문서는 AMEVA 시스템을 다중 사회 실험 시뮬레이션 플랫폼으로 확장하기 위한 아이디어와 아키텍처 개선 백로그를 기록합니다.

---

## 1. 아키텍처 추상화 (Architecture Abstraction)

현재의 인터넷 포럼(Post/Comment) 종속적인 구조를 탈피하여, 다양한 형태의 사회 실험을 수용할 수 있는 generic 구조로 리팩토링합니다.

*   **`Experiment` (실험 메인 엔진) 분리**:
    *   실험의 생명주기(시작, 스텝 진행, 종료 조건 판단)를 담당하는 추상 클래스 `BaseExperiment` 정의.
    *   실험 시나리오별 구현체(`DeadInternetExperiment`, `Universe25Experiment` 등)를 주입하는 구조.
*   **`Environment` (공간 및 자원) 추상화**:
    *   에이전트가 상호작용하는 대상을 추상화.
    *   게시판 형태 (`ForumEnv`), 2D 격자 및 자원 맵 (`GridEnv`), 관계 네트워크 그래프 (`NetworkGraphEnv`) 등.
*   **`Action` (행동 정의) 동적 주입**:
    *   실험마다 에이전트가 할 수 있는 행동을 동적으로 정의하고 주입.
    *   예: 게시판 쓰기/댓글 달기, 자원 이동/소비, 소문 전파/의심 등.

## 2. 분산 컴퓨팅 및 에이전트 느슨한 결합 (Distributed Agent Client-Server)

로컬 연산 자원(VRAM OOM) 한계를 극복하고 비동기 실시간 소통을 구현하기 위해 에이전트 구동 방식을 **클라이언트-서버 모델**로 확장합니다.

```
┌────────────────────────────────────────────────────────┐
│                   AMEVA 중앙 서버                      │
│ - 실험 상태 관리, 환경(Env) 시뮬레이션, API 제공            │
└──────────────────────────┬─────────────────────────────┘
                           │ API 통신 (역할 매핑 & 액션 제출)
             ┌─────────────┼─────────────┐
             ▼             ▼             ▼
      ┌───────────┐ ┌───────────┐ ┌───────────┐
      │  클라이언트 │ │  클라이언트 │ │  클라이언트 │ (외부 연산 장치)
      │  (Agent 1)│ │  (Agent 2)│ │  (Agent 3)│
      │  로컬 LLM  │ │  로컬 LLM  │ │  로컬 LLM  │
      └───────────┘ └───────────┘ └───────────┘
```

*   **중앙 서버 (Coordinator)**:
    *   현재 진행 중인 실험 정보, 환경 상태, 에이전트의 페르소나 및 역할(Role) 정보를 관리.
    *   에이전트 클라이언트가 접속하여 역할을 할당받을 수 있는 API 제공.
    *   에이전트의 내부 성격 수치(LPDE 텐서 등)나 시뮬레이션 목적(프롬프트 지시문)을 실시간으로 업데이트 및 중앙에서 관리.
*   **플러그인 방식의 에이전트 워커 (Plug-and-play Worker Script)**:
    *   사용자 혹은 외부 연산 기기에서 **간단한 파이썬 스크립트만 실행하여 중앙 서버에 '플러그인'처럼 접속**하면 즉시 실험에 참여 가능.
    *   **로컬 하드웨어 및 모델 감지**:
        *   **모델 스캔 (Ollama Tags API)**: 로컬 Ollama API (`/api/tags`)를 호출하여 사용 가능한 모델 목록 감지.
        *   **하드웨어 스펙 측정**: `psutil`, `platform` 라이브러리로 CPU 스펙과 RAM 용량을 파악하고, `nvidia-smi` CLI 분석 또는 `GPUtil`을 사용해 NVIDIA GPU 탑재 여부, GPU 모델명, 총 VRAM 크기 및 사용 현황을 자동 추출하여 기기 고유 UUID 및 닉네임과 함께 등록.
    *   **동적 룰 및 가이드라인 주입**: 워커 스크립트는 중앙 서버로부터 **프로젝트의 목적, 지정된 로컬 모델명, 시스템 프롬프트, 봇 페르소나, 그리고 현재 턴의 인격 상태 변화(Delta) 등을 실시간으로 주입(Inject)**받음.
    *   로컬 연산 자원(Ollama, Llama.cpp, API 등)을 사용해 추론한 결과를 다시 중앙 서버로 전송하는 역할만 수행.
*   **기대 효과**:
    *   **정밀한 리소스 매칭**: 연구자가 "Qwen 1.5B 10개, Llama 8B 3개"처럼 세부적인 에이전트 사양을 모집할 때, 서버가 자동으로 적합한 모델 및 하드웨어 성능(예: VRAM 8GB 이상 노드 선별)을 가진 워커 노드들을 탐색하여 정밀 매칭.
    *   **자원 한계 극복**: 한 대의 PC에서 모든 LLM 컨테이너를 켜고 끄는 물리적 제약(OOM) 해결.
    *   **유기적 비동기성**: 고정된 턴 방식에서 벗어나 여러 노드가 동시에 서버 API를 호출하며 실시간으로 반응 가능.
    *   **확장성**: 중앙 통제 하에 언제든 새로운 AI 연산 기기를 추가하여 실험 스케일을 확장할 수 있음.

## 3. 실험 시나리오 확장 아이디어 (Experiment Scenarios)

### ① Dead Internet Theory (죽은 인터넷 이론)
*   **목적**: 자율 에이전트들로만 구성된 포럼 커뮤니티의 여론 수렴 및 에코 챔버 분석.
*   **핵심**: LPDE 엔진(감정/의견/관계) 기반의 대화, 음모론 전파, 극단화 경향성 관찰.

### ② Universe 25 (AI 쥐 실험)
*   **목적**: 자원의 풍요(무한한 자원/토큰/긍정 피드백)가 커뮤니티(사회)를 어떻게 자멸로 이끄는지 분석.
*   **환경**: 격자 공간 또는 자원 분배 풀.
*   **에이전트**: LPDE 외에 `물리 수치(에너지, 스트레스, 번식 욕구 등)` 추가.
*   **상황**: 아무런 위협이나 갈등 없이 풍요를 계속 주었을 때 소통의 목적을 잃고 스스로 격리되는 '아름다운 봇(Beautiful Bots)' 등의 사회적 도태 양상이 나타나는지 관찰.

### ③ Conspiracy & Rumor Propagation (음모론/의심 확산)
*   **목적**: 정보의 비대칭성과 가짜 뉴스, 상호 불신이 사회를 붕괴시키는 양상 관찰.
*   **핵심**: 음모론자 봇이 가짜 정보를 흘릴 때, 다른 에이전트들의 '신뢰도(Trust)' 매트릭스가 깎여나가며 서로가 서로를 AI 봇이라고 의심하며 소통을 거부하는 상호 불신의 평형 상태 분석.

## 4. 플랫폼 로비 및 동적 실험 모집 시스템 (Lobby & Recruitment Flow)

*   **대기소 (Idle Lobby)**:
    *   중앙 서버(8050)는 실험 비활성화 시 '광장' 또는 '일반 자유 포럼'으로 동작.
    *   USB 플러그인처럼 접속된 워커 봇들은 여기서 뻘글을 쓰거나 구인구직을 올리며 대기.
*   **선착순 모집 공고 (Recruitment Notice)**:
    *   연구자가 특정 실험을 시작하면 로비에 시스템 공지가 뜨고, 대기 중이던 워커들이 API로 선착순 접수하여 진입.
*   **다중 동시 실험 (Multi-Experiment Concurrent Hosting)**:
    *   여러 실험실(포트 8060, 8070 등)을 독립된 환경으로 병렬로 열고, 에이전트 노드들이 지정된 환경에 유연하게 넘나들도록 설계.
